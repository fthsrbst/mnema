One memory for every AI agent.

Claude Code, Cursor, opencode, Codex CLI — each forgets when the session ends. Mnema is a local-first context layer that gives them all one common brain: authoritative context, structured memory, hybrid RAG, detailed project maps, a separate professional profile and typed relations. Run it entirely on infrastructure you control — a hosted Cloud profile is in active development.

01

Agents forget. Mnema doesn't.

problem / solution
WITHOUT A HUB
  • A decision made in Claude Code is invisible to Cursor an hour later
  • Switching desktop → laptop means re-explaining everything
  • Every agent re-derives project context from scratch
  • "I learned X yesterday" lives in scrollback — or nowhere
WITH MNEMA
  • Decisions saved once, with rationale, retrievable by any agent
  • Project maps and your professional profile follow you across connected devices
  • Relevant memory is auto-injected into each message — precisely filtered
  • Notes and research live in a searchable RAG store, forever

SQLite stays authoritative. External vector stores stay disposable. Just enough infrastructure to make context durable, explainable and portable across tools and devices.

05

Cloud — next phase

community stays free · cloud is in development
Roadmap / honest status

A hosted Mnema Cloud is in active development.

Community — the self-hosted server on this page — is the full open-source product, free today. Cloud adds accounts, organization tenancy and database-enforced isolation on top of the same core: one MCP + REST surface, one authoritative store. There is no public URL and no checkout yet.

Accounts Organization tenancy Database-enforced isolation

Planned security model: Supabase Auth, one organization per active tenant context, forced Postgres row-level security, composite tenant foreign keys, guarded Postgres RPCs, MFA for billing/destructive actions, a shared fail-closed Valkey limiter, and signed idempotent Paddle webhooks. Community — self-hosted, free, open source — is unaffected either way.

FULL INSTALL ·· paste this into your coding agent
Install and fully configure Mnema Community on this machine.

Repository: https://github.com/fthsrbst/mnema
Goal: a working local-first server, web UI, authenticated MCP endpoint, and a verified first project map.

1. Detect the OS, CPU architecture, shell, existing Node.js/Git versions, and whether port 8033 is already in use. On Raspberry Pi, explicitly detect ARM64 vs ARMv7 and treat the Pi as an always-on service host.
2. Require Node.js 22 or newer and Git. Install only missing prerequisites using the platform's standard package manager; do not replace working system packages unnecessarily.
3. If Mnema already exists, inspect its Git status and preserve user changes. Otherwise clone the repository. Prefer the official one-click installer for this OS; fall back to the documented manual install only if needed.
4. Create .env from .env.example if absent. Generate a unique HUB_TOKEN with at least 32 random bytes. Never print, log, commit, or send the token anywhere except the local service and the requested MCP client configuration.
5. Install locked dependencies, build the TypeScript server and web UI, then start Mnema. For Raspberry Pi/Linux, configure a non-root systemd service with restart-on-failure and start-on-boot. For Windows, use the documented local service path without opening a visible terminal window.
6. Verify with real requests: /health must return HTTP 200 JSON; authenticated /api/profile must return JSON rather than HTML or 404; /mcp must answer an authenticated tools/list request. Report the exact status of vector search and the FTS-only fallback if no embedding key is configured.
7. Connect the current coding agent to [HUB-URL]/mcp using the generated bearer token and the agent's supported user-level MCP configuration. Do not weaken authentication to make the connection pass.
8. Ask for the first repository path if it is not obvious. Read skills/map-project/SKILL.md, inspect that repository, create or update its grounded Mnema project map, and verify project_get returns architecture, modules, entry points, commands, data model, decisions, current focus, and next steps.
9. If another Mnema node exists, configure least-privilege sync credentials and prove both replicas converge by comparing authoritative record counts. Never copy SQLite databases through Git.
10. Finish with evidence: service state, local UI URL, MCP URL, health JSON summary, authenticated profile result, tool count, mapped project name, startup behavior after reboot, and anything still unproven.

Stop and explain the blocker instead of claiming success if any live verification fails.

# Cursor / Windsurf / opencode / Codex CLI: same endpoint, see README

Raspberry Pi is a first-class host: use it as a quiet, always-on context appliance while your workstation handles local models and media generation. Community also runs on a workstation, home server, VM or container — the host is your choice. No GEMINI_API_KEY? The server still runs and falls back to keyword-only (FTS) search instead of hybrid. Nothing crashes; you lose semantic recall until you add a key. Embeddings are only ever computed server-side.
